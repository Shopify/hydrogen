CORE · Cart drawer reference

This page freezes every state of the app-owned cart drawer: empty, a single line, multiple lines, an in-drawer quantity control, a sale / compare-at line, and the updating / removed / error live-region states. The live overlay drawer (with its native Invoker-Commands open/close wiring) lives in the shared header above; the panels below frame that same drawer markup inline so each state can be read side by side. Every panel reuses the real <s-cart> behavioral hooks (data-partials, data-removed-message, data-updated-message, data-update-error) and the cart copy from content.json. See notes/cart.md for the dynamic contract.

Empty cart

No lines, no count badge, no totals footer. Uses cart.empty / cart.emptyDescription; the only action is to continue shopping.

Updating

While a line quantity is in flight: the affected line is dimmed and marked aria-busy="true", interactive controls are disabled, and the polite live region (data-cart-status) carries cart.updated ("Cart updated") from the data-updated-message hook.
